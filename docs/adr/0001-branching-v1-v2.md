# ADR 0001: 브랜칭 v1-v2

## 상태

수락됨

## 배경

v1과 v2를 명확히 분리하여 병렬 유지보수 및 점진적 마이그레이션이 필요하다.

## 결정

* **v1**: `v1` 브랜치 + `v1.0.0` 태그로 고정. 배포/운영 기준선.
* **v2**: `v2` 브랜치에서 모든 v2 개발 진행.
* v1 핫픽스: `v1`에서 분기 후 수정, 태그 갱신.

## 결과

* v1/v2 코드베이스가 물리적으로 분리됨.
* v2는 "구현보다 문서가 먼저" 원칙 하에 문서 세트(FRD/NFR/ADR)를 먼저 커밋한 뒤 구현을 진행함.
