# ADR-0002: 싱글 테넌트(1인 전용) 운영 모델

- Status: **Accepted**
- Date: 2026-02-14

## Context
everbit v2는 개인 사이드 프로젝트이며, 1인 1계정만 사용한다.  
멀티 테넌트/권한/과금/온보딩을 v2 MVP 범위에 포함하면 복잡도와 보안 표면이 급증한다.

## Decision
- 사용자 모델은 **OWNER 단일**로 고정한다.
- 인증은 **카카오 OAuth2 단일**이며, 최초 로그인 사용자가 OWNER가 된다.
- 이후 로그인은 OWNER 카카오 계정만 허용하고, 다른 계정은 차단한다.

## Consequences
- MVP 개발 속도가 증가한다.
- 멀티 유저 기능(공유/권한/계정 관리)은 v2 범위 밖으로 명확히 밀린다.
- 운영/보안 정책은 “단일 사용자” 가정 하에서 단순화되나, 시크릿/실거래 위험은 동일하므로 보안은 완화하지 않는다.

## Alternatives Considered
1) 초기부터 멀티 테넌트 지원  
- 단점: 인증/권한/데이터 격리/감사로그 등 필수 요소가 증가하여 MVP가 지연됨.
